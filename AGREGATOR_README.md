# Aggregator Agent - Центральный агент агрегации сигналов

## Описание

**AggregatorAgent** - это центральный агент системы, который получает сигналы от всех остальных агентов, объединяет их, приоритизирует и формирует финальные рекомендации для пользователя.

## Функциональность

### 1. Сбор сигналов
AggregatorAgent получает сигналы от всех агентов через EventRouter:
- **MarketAgent**: тренды, уровни поддержки/сопротивления, волатильность
- **OnChainAgent**: движения китов, whale транзакции
- **LiquidityAgent**: зоны ликвидности, стоп-кластеры
- **ShitcoinAgent**: потенциал пампа/дампа щиткоинов
- **EmergencyAgent**: срочные сигналы (urgent buy/sell/exit)

### 2. Обработка и приоритизация

#### Веса сигналов по типам:
- **Emergency сигналы**: 0.8-1.0 (самые важные)
- **Market сигналы**: 0.4-0.7
- **OnChain сигналы**: 0.8-0.9
- **Liquidity сигналы**: 0.6-0.8
- **Shitcoin сигналы**: 0.3-1.0 (зависит от типа)

#### Веса приоритетов:
- **CRITICAL**: 1.0
- **URGENT**: 0.9
- **HIGH**: 0.7
- **MEDIUM**: 0.4
- **LOW**: 0.2

#### Консенсус бонус:
Если несколько агентов дают одинаковый сигнал, добавляется бонус до 20% к уверенности.

### 3. Формирование финального решения

AggregatorAgent генерирует одно из действий:
- **BUY** - сигнал на покупку
- **SELL** - сигнал на продажу
- **EXIT** - срочный выход из позиции
- **WAIT** - ожидание более четких сигналов

Каждое решение включает:
- **Уровень уверенности** (0-100%)
- **Уровень риска** (Low/Medium/High)
- **Список причин** (до 5 основных факторов)
- **Ценовые уровни** (Entry, SL, TP если доступны)

### 4. Логирование

Все агрегированные сигналы сохраняются в SQLite:
- Исходные сигналы от агентов
- Финальные решения
- Уверенность и риск
- Причины сигнала

### 5. Telegram уведомления

AggregatorAgent отправляет структурированные сообщения в Telegram:

```

BTCUSDT @ 42153.25

📊 Уверенность: 85.3%
🟡 Риск: Medium

  • Пробой уровня 41,900
  • Всплеск объема +3.2x
  • Кит активность $1,250,000
  • Имбаланс стакана 15.2% (покупка)

Уровни:
  📍 Entry: 42153.25
  🛑 SL: 41731.72
  🎯 TP: 42785.55

💡 Рекомендация: Вход подтверждён, тренд ускоряется.

```

## Алгоритм работы

1. **Сбор сигналов** (каждую секунду)
   - Получает сигналы от EventRouter через callback
   - Хранит последние 50 сигналов на символ

2. **Агрегация** (каждые 10 секунд)
   - Фильтрует свежие сигналы (последние 60 секунд)
   - Группирует по направлению (BUY/SELL/EXIT)
   - Вычисляет скоринг для каждого действия
   - Определяет финальное действие

3. **Отправка** (если уверенность > 50%)
   - Форматирует сообщение
   - Отправляет в Telegram
   - Сохраняет в БД

4. **Периодические отчеты** (каждый час)
   - Генерирует сводку активности
   - Отправляет в Telegram

## Настройка

### Пороги уверенности

В `aggregator_agent.py` можно настроить:
- `min_confidence = 0.5` - минимальная уверенность для отправки
- Интервалы агрегации (по умолчанию 10 секунд)
- Время хранения сигналов (по умолчанию 60 секунд)

### Веса сигналов

Веса можно настроить в `self.signal_weights`:
```python
self.signal_weights = {
    "emergency": {
        "price_spike": 1.0,  # Изменить вес
        ...
    },
    ...
}
```

1. Добавьте вес в `self.signal_weights`
2. Обновите логику в `_aggregate_signals()` если нужно
3. Обновите `_extract_reasons()` для извлечения причин

## Интеграция

AggregatorAgent автоматически интегрирован в систему через:
- **EventRouter**: получает все сигналы через callback
- **TelegramBot**: отправляет финальные решения
- **Database**: сохраняет историю

## Производительность

- Обработка сигналов: асинхронная
- Задержка агрегации: 10 секунд
- Задержка отправки: < 1 секунда
- Память: ~1MB на 1000 сигналов

## Расширение

Для добавления новых источников данных:
1. Создайте новый агент
2. Отправляйте сигналы через `event_router.add_signal()`
3. AggregatorAgent автоматически получит их
4. При необходимости добавьте веса в `signal_weights`
